# BLAS vs Sequential Threshold Analysis

**Date:** 2026-04-09
**Machine:** 166-core devserver (83 physical cores), L3 32MB
**Build:** `@fbcode//mode/opt`

## Summary

The BLAS/sequential crossover for IndexFlatL2 depends on `nq * d` and `omp_get_max_threads()`.
The variable `nb` (database size) does NOT affect the crossover point — both paths scale
linearly with nb at the same rate.

## Methodology

- IndexFlatL2 with L2 distance
- Each config: 1 warmup + 5 timed runs, averaged
- Force BLAS: `distance_compute_blas_threshold = 1`
- Force Sequential: `distance_compute_blas_threshold = INT64_MAX`
- Wall clock time (includes thread overhead)
- Ratio = Seq_time / BLAS_time. >1.0 means BLAS is faster.

|---------|---------|---------|
| 1 | 640 (1.13x) | 640 (1.20x) |
| 10 | 32,000 (1.42x) | 32,000 (1.24x) |
| 40 | >64,000 | ~64,000 (1.09x) |
| 80 | >64,000 | >64,000 (0.87x) |
| 166 | >64,000 | ~64,000 (0.90x) |

### nb effect explanation

At large nb (100M+), the working set (nb * d * 4 bytes = 24-48GB) far exceeds
L3 cache (32MB). BLAS's tiled sgemm has better cache locality than the sequential
per-query scan, giving BLAS a ~2x relative advantage. This shifts the crossover
earlier by roughly one nq step.

The effect is most pronounced at high thread counts where sequential's parallelism
advantage is already diminished. At OMP=1, the crossover barely changes because
BLAS already dominates at small nq*d.

## Key Findings

1. **Thread count is the dominant factor.** The crossover scales roughly linearly
   with threads up to ~20 threads, then flattens significantly.

2. **nb has a moderate effect at large scale.** At nb=100M, the crossover shifts
   ~2x earlier compared to nb=100K due to cache effects. Both paths scale with nb,
   but BLAS's tiled approach handles cache-busting working sets better.

3. **At OMP=1, BLAS wins at surprisingly small nq*d (~640-1280).** This is the
   OVIS regression case. MKL's internal threading gives BLAS an advantage even
   for single-query searches.

4. **At high thread counts (80+), sequential dominates for d=64-128.** With 166
   threads and d=128, sequential wins up to nq*d ~64K (with large nb).

5. **Thread scaling is sublinear.** The crossover scales ~linearly up to 20 threads,
   then flattens. At 166 threads, the effective crossover is only ~50-100x the
   OMP=1 crossover, not 166x.
